Web Storage

  • The data in storage is locked to a single origin. Meaning that you cannot access data that was stored from a different page. Only data stored from your own page.
  • Web storage uses a key / value storage system. When you save a piece of data you supply a key and a value. When you want to retrieve the data you provide the same key used to store it, and you will get back the value.
  • There are 2 different mechanisms for storing data. Session Storage, and Local Storage
  • Session Storage will keep the data around as long as the browser remains open, even if the user reloads the page.
  • Local Storage will persist through the browser opening and closing. Even after closing the browser the next time it is opened the data will still be accessible.
